Data Structure and Algorithms


Data Structure:

Data Structure are the way to arrange data. so that data items can be used efficiently in memory.

Data Structure are way in which data are stored for efficient search and retrieval.


Algorithm:

An Algorithm is a finite set of instructions or logic, written in order, to accomplish a certain predefined task.

Note:

Algorithm is not the complete code or program. is just core logic (solution) of a problem.

which can be expressed eaither as an informal high level description as pseudocode or using a flowchart.


Every algorithm must satisfy the following properties:

  1. Input:

    2. There should be 0 or more inputs supplied externally to the algorithm.

  2. Output:

    3. There should at least one output to be obtained.

  3. Definiteness:

    4. Every step of algorithm should be clear and well defined.

  4. finiteness:

    5. The algorithm should have finite number of step.

  5. Correctness:

    6. Every step of the algorithm must generate a correct output.


An algorithm is said to be efficient and fast, if it takes less time to execute and consumes less memory space.

The performance of an algorithm is measured on the basis of following properties:

  1. Space complexity
  2. Time Complexity  will added soon

Memory layout of C program

When we run the program, it's executable file is stored in the RAM of the computer in an organized manner.

To understand the way our C program works, we need to understand the arrangement of the memory assigned to our parogram.

All the variables, functions and data structures are allocated memory into special memory segment is mainly divide into four different parts which are specifically allocated to different types of data defined in our program


1] Code Segments:

This include the current activity represented by the value of program counter and the contents of the processor's register. this defines an area in memory that store the instruction codes. this is also a fixed area.

2] Stack:

The stack hold the temporary data such as method/function parameters, return address and local variabels. the return address refers to the address to return after completion of function execution.
This segment size is variable as per local variable, function parameters and function calls. this segment grows from a higher address to lower address.

3] Heap:

Heap contains data which is requested by the program as dynamic memory. this is area of memory allotted for dynamic memory storage such as for malloc(), and calloc() calls.
this segment size is also variable as per user allocaion. this segment grows from lower address to higher address.

4] Initialised Data:

This is the portion of the object file or program's virtual address space that consists of un-initialised static and global variable.

5] Un-initialised Data

This is read-write, since the values of variable could be changed during run time. this segment also has fixed size. un-initialised data segment is also called BSS(Block Started By Symbol) segment
