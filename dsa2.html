Space Complexity

Space complexity is the total amount of memory space used by an algorithm/program including the space of input values for execution. to find space complexity, we need to calculate the space occupied by the variables used is an algorithm or program. 

   
   Space complexity = Auxiliary space + Space used by input values

Auxiliary space is just a temporary area or extra space and it is not same as space complexity.

Note:

    The best algorithm/program should have the less space complexity, the lesser the space used, the faster it executes.

Calculating Space Complexity:

Let us understand the Space-Complexity calculation through examples.

Example 1: 
#include <stdio.h>

int main(){
  int a, b, c;
  a = 5, b = 5;
  c = a + b;
  printf("%d",c);
  return 0;
}
Output:
  10

In the above program three integer variable are used. assume the size of each integer variable is 4 bytes. So, the total space occupied by the program is 4 * 3 = 12 bytes. since no additional variable are used , no extra space is required.

Space complexity for the above program is O(1) or constant.


Example 2:

include <stdio.h>
int main()
{
  int n, i, sum = 0;
  scanf("%d", &n);
  int arr[n];
  for(i = 0; i < n; i++)
  {
    scanf("%d", &arr[i]);
    sum = sum + arr[i];
  }
  printf("%d", sum);
  return 0;
}
Output: 
  10

In the above program, the array hold n integer elements. So, the space occupied by the array is 4 * n bytes. Also we have integer variables such as n, i and sum. Assuming 4 bytes for each variable, that is 4 * 3 = 12 bytes. the total space occupied by the program is 4n + 12 bytes. Since the highest order of n in the equation 4n + 12 is n, so the space complexity is O(n) or linear.


Big O Notation Space Complexity
O(1) Constant Space Complexity occurs when the program doesnâ€™t contain any loops, recursive functions or call to any other functions.
O(n) Linear space complexity occurs when the program contains any loops or recursive function.
